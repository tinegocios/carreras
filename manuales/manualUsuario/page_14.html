Ingreso de Datos GPS
Múltiples métodos de recepción incluyendo HTTP POST, WebSocket y sentencias NMEA crudas con registro completo

Vía HTTP API

Método principal para la ingesta de datos GPS a través del endpoint HTTP POST /api/gps.

  • Recibe datos procesados y formateados en JSON
  • Ofrece robustez y desacoplamiento entre componentes

Vía WebSocket

Conexión WebSocket directa desde el dispositivo al backend para comunicación de baja latencia.

  • Comunicación bidireccional
  • Ideal para envío de comandos al rover en tiempo real

Vía NMEA Crudo

Puerto TCP/WebSocket dedicado para recepción de sentencias NMEA crudas.

  • Replica la funcionalidad del servidor Python
  • Reduce el número de componentes intermedios

Registro y Logs

Independientemente del método de ingreso de datos utilizado, el backend registra todas las posiciones recibidas.
